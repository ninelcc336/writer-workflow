# 章节规划说明

本文件把 `plan-chapter` 定义成一套可直接执行的流程。

目标只有一个：

- 在不写正文的前提下，产出一份可供人审核的章节施工图

这份施工图必须让人能快速判断：

- 这一章要推进什么
- 怎么推进
- 容易写崩的地方在哪里
- 结尾准备把读者带到哪里

## 一、触发条件

满足以下条件后，才允许执行 `plan-chapter`：

- 新书初始化已完成
- `book/canon/` 具备最小基础文件
- `book/state/` 已脱离模板占位状态
- 用户明确要求规划某一章，或允许进入下一章规划

## 二、禁止前提

有以下任一情况时，不得执行 `plan-chapter`：

- `book/canon/setting.md` 仍缺失
- `book/state/characters.yaml` 仍是模板占位
- 上一章尚未完成最终审核，但要强行规划下一章
- 必要的章节输入还不存在

## 三、最小输入

执行 `plan-chapter` 时，至少需要这些输入：

1. 当前书的 canon 基础信息
2. 当前结构化 state
3. 上一章定稿或上一阶段收束信息
4. 当前章 brief

如果是第 1 章，上一章输入可为空，但必须有：

- `book/canon/premise.md`
- `book/canon/setting.md`
- `book/canon/style_rules.md`
- `book/canon/characters/protagonist.md`
- `book/canon/volumes/volume-01-outline.md`

## 四、建议读取顺序

建议 agent 按以下顺序建立上下文：

1. `book/canon/premise.md`
2. `book/canon/setting.md`
3. `book/canon/style_rules.md`
4. `book/canon/characters/protagonist.md`
5. 当前卷纲
6. 当前章 brief
7. `book/state/*.yaml`
8. 上一章 final 或上一章状态差异

## 五、执行步骤

### 步骤 1：确认章节输入

先确认以下问题：

- 这章是第几章
- 属于哪一卷
- 本章 brief 是否存在
- 上一章的结尾状态是否清楚

如果其中任一项不清楚，先补足输入，不要硬做规划。

### 步骤 2：提取本章必须处理的内容

至少提取：

- 本章核心目标
- 必须兑现或推进的伏笔
- 必须维持的一致性约束
- 本章可用人物与资源
- 本章的预期情绪或爽点

### 步骤 3：拆成节拍

将本章拆为 3 到 5 个节拍。

每个节拍至少要包含：

- 节拍名称
- 节拍功能
- 预估字数
- 关键动作
- 情绪或落点

要求：

- 节拍之间要有明显功能分工
- 不允许所有节拍都只是重复推进
- 要有明确的收束节拍

### 步骤 4：给出字数预算

整章必须有目标字数。

预算规则：

- 核心爽点或核心冲突节拍占最大预算
- 过渡节拍应受到控制
- 章末钩子节拍不应无上限膨胀

如果节拍预算加总明显高于目标字数，必须先修正计划，不能把失控留给正文阶段。

### 步骤 5：列出必写节点

用清单写明本章不允许遗漏的内容。

这些内容应该足够具体，例如：

- 某角色必须第一次出场
- 某伏笔必须推进到什么程度
- 某装备或规则必须第一次被使用

不要写成空话，例如“本章要精彩”“本章要有爽点”。

### 步骤 6：列出风险与禁忌

至少写出：

- 最容易写崩的人设点
- 最容易打架的设定点
- 最容易失控的字数点
- 最容易套路化的收尾方式

### 步骤 7：确定章末落点

必须明确本章结尾属于哪一类：

- 钩子推进
- 爽点承诺
- 情绪收束
- 冷收束

不能留成模糊状态。

### 步骤 8：输出章节计划

建议输出到：

- `book/drafts/chapter-XXX/plan.md`

同时建议生成一份检查报告：

- `book/artifacts/reports/chapter-XXX-plan-check.md`

## 六、计划文件必须包含的内容

一份合格的章节计划至少要有：

1. 章节基础信息
2. 本章目标
3. 字数预算
4. 3 到 5 个节拍
5. 必写节点
6. 风险与禁忌
7. 章末落点
8. 涉及的关键 state 约束

## 七、计划检查项

在进入人审前，agent 至少应自查：

- 本章是否真的推进了某个目标
- 节拍预算是否合理
- 必写节点是否足够具体
- 是否遗漏关键伏笔
- 是否和当前 state 冲突
- 章末落点是否清楚

## 八、计划阶段禁止事项

在 `plan-chapter` 阶段，不允许：

- 直接写正文段落
- 提前更新 `book/state/*.yaml`
- 把计划当成定稿
- 用“写的时候再看”跳过风险项

## 九、完成标准

满足以下条件时，可视为 `plan-chapter` 完成：

- 章节计划文件已生成
- 计划检查报告已生成
- 用户能据此直接审核是否允许进入正文起草

只有在用户确认通过后，才允许进入 `draft-chapter`。
