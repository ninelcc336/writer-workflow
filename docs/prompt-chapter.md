# 正文骨架说明

本文件把 `prompt-chapter` 定义成一套可直接执行的流程。

目标只有一个：

- 根据已经通过审核的章节计划，生成一份可直接喂给其他模型的正文骨架 / 提示词文件

这个阶段的产物不是正文，而是“让模型知道该怎么写这一章”的标准输入文件。

## 一、触发条件

满足以下条件后，才允许执行 `prompt-chapter`：

- `plan-chapter` 已完成
- 用户已明确通过章节计划
- 当前章 `plan.md` 存在
- 当前书的 canon 与 state 可正常读取

## 二、禁止前提

有以下任一情况时，不得执行 `prompt-chapter`：

- 当前章计划尚未通过人审
- 当前章 `plan.md` 缺失
- state 仍明显不完整
- 当前章关键输入仍是模板占位

## 三、最小输入

执行 `prompt-chapter` 时，至少需要这些输入：

1. `book/canon/` 基础文件
2. `book/state/*.yaml`
3. 当前章 `plan.md`
4. 当前章 brief
5. 上一章 final 或上一章状态收束

## 四、产物定位

`prompt-chapter` 的输出文件应同时满足两类用途：

1. 你可以直接拿去喂其他模型
2. 本仓库里的 `draft-chapter` 也可以直接消费它

因此它必须是：

- 信息完整的
- 指令结构清楚的
- 不依赖隐含上下文的

## 五、执行步骤

### 步骤 1：读取计划和上下文

优先读取：

- `book/canon/premise.md`
- `book/canon/setting.md`
- `book/canon/style_rules.md`
- 当前章 `plan.md`
- `book/state/*.yaml`
- 当前章 brief

### 步骤 2：抽取提示词所需信息

至少抽取：

- 章节标题
- 本章目标
- 目标字数
- 节拍拆分
- 必写节点
- 章末落点
- 关键 state 约束
- 风格要求
- 关键角色
- 关键势力
- 关键资源 / 能力
- 必须推进的伏笔
- 禁忌项 / 风险项

### 步骤 3：组装正文骨架

正文骨架至少应包含这些部分：

1. 角色或写作人格设定
2. 作品定位与本章任务
3. 前情提要
4. 必须调用的连续性信息
5. 本章节拍
6. 必写节点
7. 风格规则
8. state 约束
9. 禁忌项
10. 输出格式要求

### 步骤 4：输出提示词文件

建议输出到：

- `book/drafts/chapter-XXX/prompt.md`

如果需要，也可补一份结构化元数据：

- `book/drafts/chapter-XXX/prompt.data.yaml`

## 六、提示词文件必须包含的内容

一份合格的 `prompt.md` 至少要有：

1. 章节基础信息
2. 作品定位与本章任务说明
3. 前情提要
4. 关键角色 / 势力 / 资源 / 伏笔
5. 节拍清单
6. 必写节点
7. 风格要求
8. state 约束
9. 禁忌项
10. 结尾要求
11. 输出格式要求

## 七、禁止事项

在 `prompt-chapter` 阶段，不允许：

- 直接写正文内容
- 提前更新 `book/state/*.yaml`
- 省略关键约束，导致提示词无法脱离上下文单独使用

## 八、完成标准

满足以下条件时，可视为 `prompt-chapter` 完成：

- `prompt.md` 已生成
- 内容足以单独提供给其他模型
- 本仓库内的 `draft-chapter` 可以直接消费它
